S13 Mapboard Selection & Features

13. MAPBOARD SELECTION & FEATURES

 

13.1  PROCEDURE: The following rules are used to determine each mapboard for a SASL Mission. Consult the Mission's "Mapboard Selection & Activation" section for specifics on when and how each mapboard for that Mission is activated. Some Missions allow additional mapboards to be activated during play, usually due to a FRIENDLY unit entering a new Location near the current EBE.

13.2  SELECTION PROCESS: Perform the following steps (13.21-.2312, as necessary) for each mapboard randomly generated:

13.21  DETERMINE TERRAIN COLUMN: Make a DR on the proper subtable (a-e) of Table {A8}. The result gives a Terrain Column to be used on Table {A9}. The "Mapboard Selection & Activation" section of each Mission lists the proper subtable of {A8} to use for that Mission.

13.22 DETERMINE MAPBOARD: Make a DR (A9) using the Terrain Column determined from Table {A8}. The result is the board number of the just activated map. Reroll, as necessary, if the determined mapboard is not available or is currently in play.7

13.23 MAPBOARD ALIGNMENT: Make a random dr to determine each mapboard's alignment. On a dr of  ≤ 3 the board number is placed towards the north. If necessary, place S?/VPO Control markers as dictated by the Mission being played (4.la-e, 14.1).

13.231 ROAD NETWORK: A MSR may require that the mapboards generated be placed such that a continuous path of "connected" road hexes exists from one board edge to another (usually from the FBE to the [current] EBE). As each mapboard is placed in such a Mission the player must insure that a continuous network of contiguous road hexes exists from/to the specified board edges. Where > one path exists (e.g., due to a "loop", or the road leading off an eligible board edge at > one Location), only the road hexes comprising the shortest (measured in MP) path are considered to be part of the Road Network. If an equidistant path is formed all road hexes of each equidistant "branch" are part of the Road Network.

If a path of contiguous road hexes does not exist when a new map is placed, reverse the alignment of the just-placed board. If a Road Network is still not formed discard that board and determine a new one until an acceptable mapboard is placed.

13.2311 ROAD NETWORK BRIDGE: Each single hex bridge along the Road Network is automatically intact (the {A9c} dr for such is ignored). If a river/canal mapboard is generated (e.g., board 7, 8 or 40) a multi-hex bridge automatically exists along the Road Network (reversing alignment as necessary; or use table {A9a} if > one hexrow is eligible).  Roll for bridge type normally {A9b/c/d}.

13.2312  PLOWED ROADS: A MSR required Road Network is considered plowed (E3.723, E3.733-.7331) if Deep Snow is present.

13.2313 PTO TERRAIN: A MSR required Road Network exists normally if PTO Terrain is in effect [EXC: a Road Network bridge exists only if the bridge is required by the Mission].

13.24  TERRAIN ALTERATIONS: The player is free to use published/homemade terrain overlays, or Broken Terrain rules (F13.1) to alter a mapboard's printed features, as desired. Steppe Terrain (F13.2) can also be used with mapboards 26-31 and various desert overlays, at the player's discretion. Use of any overlays may, however, affect both S? placement and the VPO determination processes (14.1).

13.25  FACTORY: Make a dr for each building of ≥ 7 hexes on each "City" mapboard (marked with the "c" prefix on Table {A9}). An Original dr of ≤ 3 results in the building being a Factory (B23.74).

13.3  DUPLICATION: If a board that is already in play is activated again reroll for a different board using the same {A9} Terrain Column.

13.4  STREAMS AND WATER OBSTACLES: When a mapboard is activated that has a stream/river present, the proper Chapter B tables (B20.4; B21.121-.122) should be used to determine the water's depth/current. As necessary, make a Random dr to determine the current's direction.

13.5 ONE-HEX BRIDGES: Each one-hex bridge must have a dr made {A9c} to determine if the bridge is intact or destroyed [EXC: 13.2311]. Regardless of the result make a subsequent dr {A9d} to determine the bridge's construction. If the Table {A9c} result is "Destroyed" place the appropriate rubble counter (as necessary; B6.331) at the Depression level of the hex. See also 14.12.

13.6  RIVER BOARD: When a river/canal mapboard (e.g., 7, 8 or 40)8 is activated use tables {A9a-b} to determine the presence and construction of a multi-hex bridge. A dr {A9a} of ≥ 5 results in no bridge being present; you have, in effect, reached a "dead end" [EXC: 13.2311]. If the presence of a bridge is established a subsequent dr is made {A9b} to determine the type of bridge. See also 14.12.

13.61 FRIENDLY RIVER ASSAULT: If a bridgeless river mapboard is the first non-FRJENDLY setup board selected in a Mission in which the ENEMY is in Hold Attitude (and the FBE is along the 22" length of a mapboard), the FRIENDLY side automatically receives a number of Small Rafts sufficient to transport his entire OB, including later-arriving (RE) reinforcements. A Large Raft is granted, as necessary, to transport each eligible FRIENDLY Gun (E5.122). If the FRIENDLY side is German, Assault Boats (E5.11) are received instead of Small Rafts on a subsequent dr of ≤ 3. FRIENDLY units are not subject to Command Control while on a boat in a Water Obstacle hex.

13.7  PTO TERRAIN: PTO Terrain (G1.) is possible if any non-City mapboards (i.e. Village, Farmland, Rural, or Hills) are selected for Missions in the PTO (12.23). As applicable, PTO Terrain applies to all mapboards. MSR-designated terrain takes precedence if in conflict with G1.  For Missions in the PTO, prior to Mission setup, determine if PTO Terrain (G1.) is in effect via a single DR:

13.71 PTO MODIFICATIONS: The following apply to Missions that occur when PTO Terrain (G1.) is in effect:

   •      Roads exist normally on all maps with the "c" City prefix

   •      All roads exist as Dirt Roads on all maps with the "v" Village prefix

13.8 SEABORNE ASSAULTS: Unless specified by MSR, the following apply to any Mission in which Ocean/Beach overlays are in play and one side is conducting a Seaborne Assault (G14.)

13.81  BOARDS/OVERLAYS: Beach and OCEAN overlays are used when a Seaborne Assault (G14.) is being conducted by either side. If the FRIENDLY (ENEMY) side is conducting the Seaborne Assault the Beach and OCEAN overlays are placed on the FBE (EBE) side of the boards.  The Beach overlay placement is dependent on the board configuration. For a North-South orientation (typically Configuration A), Beach overlays are placed as follows: Be4 with hexes 471-478 on FF0-GG1 (or B10-A10), Be5 with 571-578 on 406-413, and Be6 with 601-607 on 506-K1 (or W10). For an East-West orientation (typically Configuration B), Beach overlays are placed as follows: Be1, Be2 and/or Be3 with hexes 101-102, 201-202 and/or 301-302 on GG1-GG2 (A10-A9) as needed to completely cover the first hexrow of the FBE (or EBE as appropriate) with beach overlays. See G13.82 for beach width and OCEAN overlay placement. Effluent overlays may be used to connect streams to the OCEAN.

13.82  BEACH CONDITIONS: Determine the presence of a reef, beach slope, beach width, tide, and surf conditions (see G13.9). For an ENEMY Seaborne Assault, the tide is determined after the beach defenses, if any, are setup by dr; ≤ 3 the tide is Low; ≥ 4 the tide is High.

13.83  SAN: G14.261 applies to the SAN of the side conducting the seaborne assault.

13.84  WATERCRAFT: The side conducting a Seaborne Assault receives sufficient Watercraft to transport the assault force to the beach. A DR on the appropriate LC Table determines the available Watercraft for the assault force. If the FRIENDLY side is conducting the assault, the player may select any number of Watercraft (within the restrictions of G14.21) of any type available on a value ≥ the DR. If the ENEMY side is conducting the assault, the S?/ENEMY units embark on Watercraft of the single type determined (with 10 PP per S?) until the Watercraft is fully laden [EXC: a minimum of two Watercraft enter per turn in which S?/ENEMY units are available and allowed to enter; the S?/ENEMY units should be divided as evenly as possible among the available Watercraft].
